テンプレ方式 / パターン化 廃案
  • 順序テンプレを先に作る → 論点をはめ込む
  • 候補タブ切替 / 二項対比 / フロー型 の3種
  • トレードオフブランドは揃うが、毎回の論点が痩せる
  • 構造固定 → 内容が形に合わせて変形
都度生成 / 内容駆動 採用 ★
  • 順序記事の論点を見る → 図解の形を作る
  • 担保白黒3色 + 1.5pxボーダー + 2書体だけ固定
  • トレードオフブランドは揺れ得るが、論点は活きる
  • 構造自由 → 形は内容に合わせて変形
担保する3つの最小ルール

ブランド一貫性はここだけで守る

RULE 01 / カラー
3色のみ・白黒反転
アクセント色禁止、強調は背景反転
RULE 02 / 線
1.5px ソリッド
破線は警告枠のみ、装飾線は不使用
RULE 03 / 書体
Plex Mono + Noto Sans JP
見出しMono、本文Sans、混在は禁則

04 選択

STUDIO と GitHub Pages ── 役割の違う2層を共存させる

STUDIO / 通常版 読む層
  • HTML/CSS/JS制限的、Embedのiframeのみ
  • 編集Cmd+V で md を貼るだけ
  • サイト統合adaptdesign.jp 配下
  • コスト有料プラン込み
GitHub Pages / インタラクティブ版 触る層
  • HTML/CSS/JS完全自由、JS がそのまま動く
  • 編集git push (Web UI でも可)
  • サイト統合別 URL、相互リンクで接続
  • コスト無料 (Public リポジトリ)

リポジトリの3要素

FILE 01 / 触らない

FILE 02 / 毎回編集

DIR 03 / 毎回追加
{slug}/index.html
記事ごとのフォルダ + 中に index.html。Visual Article Builder の出力を配置
資産の分離理由

なぜ Public リポジトリを切り出したか

PRIVATE / 既存
r2r2r-content
記事資産 (md / 図解原稿)。Free plan で Pages は使えない
PUBLIC / 新設
interactive-articles
公開HTMLのみ。Pages 有効、原稿と公開物を分離
Settings → Pages

GitHub Pages 有効化フロー

  1. 01
    リポジトリ Settings → Pages を開く +
    SETUP Public リポジトリのみ可
  2. 02
    Source: Deploy from a branch を選択 +
    SOURCE Actions ではなく branch 配信
  3. 03
    Branch: main / (root) を選択 +
    BRANCH サブディレクトリ不要
  4. 04
    Save → 1〜2分待つ +
    WAIT 初回ビルドのみ若干かかる
  5. 05
    https://pixelstomotors.github.io/interactive-articles/ で公開 +
    LIVE 記事URL: /{slug}/ STUDIO と slug を揃える
08 運用

articles.json で動的化 ── 3記事目で破綻しない設計

初版 / 直書き構成 廃案
  • 構造index.html に記事カードを HTML 直書き
  • 追加記事ごとに index.html を編集
  • 限界3記事目で疲れる、100記事目では破綻
  • 負債初期の運用設計が10ヶ月後に効いてくる
最終 / JSON 分離 + JS 動的描画 採用 ★
  • 構造articles.json + index.html (fetch で読込)
  • 追加JSON 配列の先頭に1ブロック追記のみ
  • 挙動day 降順で自動ソート、件数カウンタも自動
  • 永続性100記事目でも持つ運用構造
